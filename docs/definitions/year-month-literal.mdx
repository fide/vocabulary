## Definition

A calendar year and month in canonical `YYYY-MM` form.

- **Boundaries:** Not a full date or year alone.
- **Hex Code:** `b2`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) (Broad) + [`xsd:gYearMonth`](https://www.w3.org/2001/XMLSchema#gYearMonth) (Exact)


---

## Fide ID Rules

These entity-specific constraints apply when YearMonthLiteral is used in [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/syntax#reference-type) [`YearMonthLiteral`](/vocabulary/definitions/year-month-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type

- [Reference](/fide-id/specification/syntax#reference-identifier) MUST use canonical `YYYY-MM` format without timezone ([XML Schema 1.1 Part 2: Datatypes](https://www.w3.org/TR/xmlschema11-2/#gYearMonth)).

<details>
<summary>Year-month references MUST use a four-digit year and zero-padded month.</summary>

- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-3`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03Z`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `03/2026`
</details>
