## Definition

A formalized Uniform Resource Identifier string.

- **Boundaries:** Not plain text.
- **Hex Code:** `c0`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) (Broad) + [`xsd:anyURI`](https://www.w3.org/2001/XMLSchema#anyURI) (Exact)


---

## Fide ID Rules

These entity-specific constraints apply when URILiteral is used in [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/syntax#reference-type) [`URILiteral`](/vocabulary/definitions/uri-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type


<details>
<summary>[[Reference](/fide-id/specification/syntax#reference-identifier) Type](/fide-id/specification/syntax#reference-type) [`URILiteral`](/vocabulary/definitions/uri-literal) is only valid when the [Entity Type](/fide-id/specification/syntax#entity-type) is also [`URILiteral`](/vocabulary/definitions/uri-literal).</summary>

- Valid [Entity Type](/fide-id/specification/syntax#entity-type): [`URILiteral`](/vocabulary/definitions/uri-literal)
- Invalid [Entity Type](/fide-id/specification/syntax#entity-type): [`Person`](/vocabulary/definitions/person)
- Invalid [Entity Type](/fide-id/specification/syntax#entity-type): [`TextLiteral`](/vocabulary/definitions/text-literal)
</details>

<details>
<summary>[Reference](/fide-id/specification/syntax#reference-identifier) MUST be a valid URI string in compliance with [RFC 3986](https://www.rfc-editor.org/rfc/rfc3986).</summary>

- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `mailto:alice@example.com`
- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `urn:isbn:9780140328721`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `example.com/alice`
</details>
