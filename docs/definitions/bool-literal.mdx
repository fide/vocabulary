## Definition

A true or false value.

- **Boundaries:** Not a number or string.
- **Hex Code:** `a3`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) (Broad) + [`xsd:boolean`](https://www.w3.org/2001/XMLSchema#boolean) (Exact)


---

## Fide ID Rules

These entity-specific constraints apply when BoolLiteral is used in [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/syntax#reference-type) [`BoolLiteral`](/vocabulary/definitions/bool-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type

- [Reference](/fide-id/specification/syntax#reference-identifier) MUST be `true` or `false` ([XML Schema 1.1 Part 2: Datatypes](https://www.w3.org/TR/xmlschema11-2/#boolean)).

<details>
<summary>Boolean references MUST use lowercase text values.</summary>

- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `true`
- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `false`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `True`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `FALSE`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `1`
</details>
