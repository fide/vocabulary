## Definition

A whole number.

- **Boundaries:** Not a decimal or string.
- **Hex Code:** `a1`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) (Broad) + [`xsd:integer`](https://www.w3.org/2001/XMLSchema#integer) (Exact)


---

## Fide ID Rules

These entity-specific constraints apply when IntegerLiteral is used in [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/syntax#reference-type) [`IntegerLiteral`](/vocabulary/definitions/integer-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type

- [Reference](/fide-id/specification/syntax#reference-identifier) MUST use a base-10 integer string ([XML Schema 1.1 Part 2: Datatypes](https://www.w3.org/TR/xmlschema11-2/#integer)).

<details>
<summary>Integer references MUST NOT use decimal points, scientific notation, or thousands separators.</summary>

- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `42`
- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `-7`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `4.2`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `1e3`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `1,000`
</details>
