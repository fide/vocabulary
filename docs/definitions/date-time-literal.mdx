## Definition

A specific point in time in canonical UTC `YYYY-MM-DDTHH:MM:SSZ` form.

- **Boundaries:** Not a date or time alone.
- **Hex Code:** `b0`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) (Broad) + [`xsd:dateTime`](https://www.w3.org/2001/XMLSchema#dateTime) (Exact)


---

## Fide ID Rules

These entity-specific constraints apply when DateTimeLiteral is used in [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/syntax#reference-type) [`DateTimeLiteral`](/vocabulary/definitions/date-time-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type

- [Reference](/fide-id/specification/syntax#reference-identifier) MUST use canonical UTC datetime format with a trailing `Z` ([XML Schema 1.1 Part 2: Datatypes](https://www.w3.org/TR/xmlschema11-2/#dateTime)).

<details>
<summary>Datetime references MUST use `YYYY-MM-DDTHH:MM:SSZ` format.</summary>

- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03-05T14:30:00Z`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03-05 14:30:00`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03-05T14:30:00-05:00`
</details>
