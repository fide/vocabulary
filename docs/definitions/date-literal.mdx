## Definition

A calendar date in canonical `YYYY-MM-DD` form.

- **Boundaries:** Not a datetime or time.
- **Hex Code:** `b1`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) (Broad) + [`xsd:date`](https://www.w3.org/2001/XMLSchema#date) (Exact)


---

## Fide ID Rules

These entity-specific constraints apply when DateLiteral is used in [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/syntax#reference-type) [`DateLiteral`](/vocabulary/definitions/date-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type

- [Reference](/fide-id/specification/syntax#reference-identifier) MUST use canonical `YYYY-MM-DD` format without timezone ([XML Schema 1.1 Part 2: Datatypes](https://www.w3.org/TR/xmlschema11-2/#date)).

<details>
<summary>Date references MUST use a four-digit year and zero-padded month and day values.</summary>

- Valid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03-05`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-3-5`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `2026-03-05Z`
- Invalid [Reference](/fide-id/specification/syntax#reference-identifier): `03/05/2026`
</details>
