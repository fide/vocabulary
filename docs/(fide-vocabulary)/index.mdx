The Fide Vocabulary defines stable base entity types shared across Fide identifier and protocol layers.
This specification is the canonical registry for valid entity types, valid reference types, and allowed entity-type/reference-type combinations used by [Fide ID](/fide-id) and [Fide Context Protocol](/fcp).

[Fide ID](/fide-id) uses these entity types for identifier construction, and the [Fide Context Protocol](/fcp) applies them to interoperable statement structures.

Each definition combines semantic meaning, external standard alignment, and any entity-type-specific identifier constraints that apply to an entity.

## Design Rule

Fide entity types are stable base kinds, not contextual roles.

A type should be used in a Fide ID only when it is intended to remain valid for that entity across contexts and over time.
If something could later still validly be understood as a different base entity type, it is usually better modeled as a statement pattern, qualifier, or domain convention rather than as a new Fide entity type.

This is why Fide prefers a small set of durable base types and avoids adding convenience types for application-specific roles unless they have distinct and stable identity criteria of their own.

## Fide Ecosystem Alignment

- [Fide ID](/fide-id) uses the vocabulary to encode entity type and reference type information into identifiers
- [Fide Context Protocol](/fcp) uses the vocabulary to define interoperable statement structures and entity semantics
